# 에피소드 4 「사라진 목소리」 — 디자인 스펙

날짜: 2026-07-08
상태: 설계 확정 (구현 전)

> 참고: 같은 날짜의 「달빛 만물상」 보너스 스테이지 초안은 이 문서로 **대체·폐기**되었다. (사용자 결정)

## 1. 개요

- **제목:** 사라진 목소리
- **소재:** 어머니의 비밀 — 라벨 없는 카세트 더미와 낡은 워크맨
- **장르:** 스토리 중심 방탈출. 복선을 깔고 회수하는 서사 + 인터랙티브 조작형 퍼즐
- **난이도/분량:** ep3보다 한 단계 위. 60~80분, 퍼즐 16개 + 최종 1개
- **시그니처 메커니즘:** **테이프 재생·되감기** — 같은 구간을 새 단서(클린 모드) 획득 후 재청취하면 해석이 바뀐다. 복선 회수가 메커니즘 그 자체.
- **떡밥 범위:** ep4 내부 완결 (시리즈 교차 떡밥 없음)
- **톤:** 시리즈 공통 — 비노골, 오해 → 사랑/희생의 반전

## 2. 스토리라인

### 진입 (다락 허브)
라벨 없는 카세트테이프 무더기 + 건전지가 새어 굳은 낡은 워크맨. 테이프에는 제목 대신 **세 자리 숫자만** 적혀 있다.

### 오해의 서사 (1막)
1980년대 말~90년대 초의 옛집. 어머니는 결혼 전 라디오 성우 지망생이었다. 어느 날부터 노래도 흥얼거림도 뚝 끊겼고, 그날이 공교롭게 주인공의 어린 시절 생일 무렵. 초반 단서들은 "나 때문에 꿈을 접었고, 그래서 노래를 멈췄다"는 죄책감으로 수렴하도록 설계한다.

### 진실 (2~3막 반전)
어머니는 목소리를 잃어가는 병을 앓았다. 진단받은 날(=노래가 멈춘 날), 남은 목소리를 아끼기로 결심하고 밤마다 골방에서 **아이의 남은 평생 몫의 목소리를 녹음**했다 — 자장가, 동화, 그리고 미래 생일마다 틀어줄 축하 메시지. 테이프의 세 자리 숫자는 곡 번호가 아니라 **"몇 번째 생일에 들려줄 것인가"**였다.

### 엔딩
주인공이 자신의 "올해 나이" 번호 테이프를 데크에 걸고 되감으면, 현재의 자신에게 도착하는 생일 메시지가 재생된다. 다락 귀환, 명패 「사라진 목소리 — 완료」.

## 3. 복선 → 회수 설계

모든 복선은 2회 노출 원칙: 처음 볼 때 자연스러운 오해 → 퍼즐로 얻은 규칙/단서로 재방문 시 진실.

| # | 복선 (깔리는 시점) | 첫인상 (오해 유도) | 회수 (재해석) |
|---|---|---|---|
| F1 | 테이프의 세 자리 숫자 (프롤로그) | 곡 번호? 날짜? | 미래 생일 나이 (P14→P16, 최종) |
| F2 | 테이프 곳곳의 잔기침 (초반 청취) | 그냥 잡음 | 병의 진행 — 클린 모드 재청취 시 기침 사이 멈춘 숨 (P13) |
| F3 | 달력의 동그라미들 (안방) | 오디션 날짜로 오인 | 병원 진료일 — 처방전 날짜와 일치 (P6·P10) |
| F4 | 골방 문틈의 담요·계란판 벽 | "혼자 있고 싶었나" | 방음 — 녹음실이었다 (P8) |
| F5 | 노래가 멈춘 날 = 생일 무렵 (P5 사진) | "내 생일에 꿈을 접었다" | 진단일 — 생일 선물로 첫 녹음을 시작한 날 (P10·P12) |
| F6 | 라디오 사연 엽서 뭉치 (마루) | 성우 시절 미련 | 병원 소개를 묻는 편지 초안 (2막 재열람) |
| F7 | 데크에 반쯤 감긴 채 걸린 테이프 (프롤로그) | 듣다 만 테이프 | 어머니의 마지막 미완성 녹음 (P15, 엔딩 직전) |

F2·F5·F7은 되감기 메커니즘으로만 회수된다 — 메커니즘과 서사의 결합 장치.

## 4. 공간·진행 구조

```
다락 (허브, 기존 Hub 재사용)
 └─ 카세트 더미 클릭 → 프롤로그 → 옛집으로

옛집 (ep4 본편)
├─ 마루 ep4-maru       — 시작 방. 전축·라디오·릴 데크
├─ 안방 ep4-anbang     — 화장대·자개장·달력
├─ 골방 ep4-golbang    — 잠긴 방 (roomGates). 녹음실의 진실
└─ 부스 ep4-booth      — 회상 공간. 데크에서만 진입/퇴장
```

- **릴 데크:** 마루에 고정이지만 수리(P1) 후 어느 방에서든 하단 바 버튼으로 여는 **오버레이 UI**. 이동 강제로 리듬을 죽이지 않는다.
- **골방 게이트:** P3·P6·P7 선행 + 노크 리듬(G) 재현으로 개방. `roomGates`로 저장 복귀 방어(기존 패턴).
- **부스:** 물리 공간이 아니라 회상 — P11로 복원한 오디션 테이프를 끝까지 들으면 진입.

### 3막 진행

| 막 | 플레이 내용 | 서사 상태 |
|---|---|---|
| 1막 (마루·안방) | 데크 수리, 첫 구간 청취, 달력·화장대 | 오해 형성 — F1~F6 복선 살포 |
| 2막 (골방·부스) | 방음벽·처방전·스플라이스·부스 회상 | 중간 반전 — F3·F4·F5·F6 회수 |
| 3막 (재청취) | 클린 모드 재청취 3건 + 숫자 규칙 | F2 회수 → F7·F1 회수, 엔딩 |

페이스 안전장치: 소리 계열(S)/문서 계열(D) 두 갈래 병렬 사슬로 항상 미해결 퍼즐 2개 이상 개방 유지. 힌트는 기존 2단계 시스템.

## 5. 퍼즐 설계 (16 + 최종)

원칙: 기입형 배제(ep2 재설계 원칙 계승), 규칙 발견 → 데이터 적용 2~3홉, 답은 관찰 가능한 단서 2개 이상 수렴, 잠금에는 서사적 이유.

### 1막 — 마루·안방

| # | 이름 | 유형 / 컴포넌트 | 내용 | 계열 |
|---|---|---|---|---|
| P1 | 데크 벨트 걸기 | 조작 / **BeltRouting** | 릴 데크 뒷판을 열고 고무벨트를 모터·풀리 3개에 드래그로 감는 경로 퍼즐. 방향이 틀리면 테이프가 거꾸로 도는 연출 | S |
| P2 | 라디오 주파수 | 소리 / **FrequencyDial** | 사연 엽서 귀퉁이의 주파수로 다이얼 미세 조정 → 드라마 재방송에서 어머니의 성우 예명 | S |
| P3 | 전축 회전수 | 규칙·소리 / **SpeedSwitch** | 45rpm 판을 33rpm으로 재생 → 늘어진 소리 속 숨은 낮은 목소리. "빠른 것은 느리게 들어라" 규칙의 첫 노출 | S |
| P4 | 카운터 되감기 | 조작 / **TapeDeck** | 쪽지 "자장가는 042부터" → 카운터 042로 되감기. 데크 튜토리얼 | S |
| P5 | 화장대 자개 문양 | 패턴 / **PatternRing** | 자개 꽃문양 링 3개 회전 정렬. 보상: 오디션 날 사진(뒷면 날짜 = 생일 무렵, F5 강화) | D |

## 6. 기술 구조 (기존 아키텍처 준수)

```
lib/puzzles-ep4.ts            # EP4_ITEMS + EP4_PUZZLES + EP4_CONFIG (id:'ep4', saveKey:'memory-box-ep4')
lib/ep4Tape.ts                # 테이프 구간 데이터·해금/클린모드 판정 (ep3TruthPair 분리 패턴)
components/scenes/Ep4Maru.tsx / Ep4Anbang.tsx / Ep4Golbang.tsx / Ep4Booth.tsx
components/ep4/TapeDeck.tsx   # 시그니처 오버레이 UI (카운터·되감기·재생)
components/ep4/…              # BeltRouting, FrequencyDial, SpeedSwitch, PatternRing,
                              # CalendarMatch, KnockRhythm, EqualizerBars, SpliceEditor, NumberBoard
```

- `types.ts`에 `Ep4RoomId = 'ep4-maru' | 'ep4-anbang' | 'ep4-golbang' | 'ep4-booth'`, `episode.ts` id 유니온에 `'ep4'` 추가
- `roomGates`: 골방 (requires P3·P6·P7 + G, fallback 마루). `finalPuzzles` 방별 매핑: 안방→P7, 골방→P11, 부스→P12, 마루→P16. `epilogueAt: 4` (P16이 마지막으로 풀리도록 P13~P15 선행 필수)
- 테이프 상태는 GameState 확장 없이 **solved 배열에서 파생 계산** (구간 해금 = 해당 퍼즐 solved, 클린 모드 = P9 solved) — 저장 스키마 변경 없음
- 조작형 결과는 기존 answer 문자열 규약 (예: 벨트 `'routed'`, 리듬 `'knock-ok'`, 수열 `'missing-N'`)
- 목소리는 실제 음성 없이 **자막 + Web Audio 톤 연출** (기존 audio.ts 확장, 시리즈 문법 유지)

## 7. 아트·사운드

- **팔레트:** 늦은 밤 백열등 — 갈색·호박색. 골방만 차가운 청록(진실의 방). 부스는 방송국 형광등 톤
- **호버:** 기존 확정 지침 — 이동 없이 brightness + glow만
- **사운드:** 테이프 릴 회전음·되감기 고속음·스플라이스 "틱" 등 Web Audio 합성. 클린 모드 전환 시 노이즈 필터가 걷히는 연출

## 8. 검증 계획

- Vitest: 전 퍼즐 판정, 골방 게이트 + 저장 복귀 방어, 숫자 수열 판정, 클린 모드 파생 로직, 구버전(ep1~3) 저장과의 독립성
- 수동 플레이스루: 노힌트 완주 / 전 힌트 완주, 데스크톱 + 모바일

## 9. 범위 제외 (YAGNI)

- 실제 음성 파일 (자막+톤으로 대체)
- 시리즈 교차 떡밥, ep1~3 클리어 연동 보너스
- 별점·타이머·랭킹 (폐기된 보너스 스테이지 기획의 요소)
- 테이프 구간의 자유 탐색 (해금된 구간만 접근 — 스포일러 방지)
